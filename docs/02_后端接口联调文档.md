# 物业管理系统后端接口联调文档

> 版本：V1.0  
> 适用范围：Web 管理端 + 微信小程序端（Resident / Agent / Admin）  
> 目标：让前后端、测试、产品在“接口路径 / 参数 / 权限 / 页面映射 / 联调顺序”上统一口径，直接进入开发阶段

---

## 1. 文档定位

这份文档从原总开发文档中单独拆出“接口与联调”部分，并针对你新补充的前端要求做了增强：

- **Web 端**：管理员主使用端，负责配置、导入导出、账单管理、报表、Agent 授权、券模板与发券规则。
- **微信小程序端**：住户主使用端，同时支持 Agent / 管理员轻量操作。
- **统一鉴权**：后端统一走 JWT Bearer Token，靠“角色 + 数据范围”控制访问。
- **统一账单口径**：账单主体永远是 `room`，住户只是查看/支付主体。

> 和旧接口清单相比，本版补了三类此前不够完整的能力：
> 1. Web 管理端登录接口  
> 2. Admin / Agent 页面必须用到的列表与状态查询接口  
> 3. 页面到接口的映射与联调顺序

## 2. 统一接口约定

### 2.1 前缀与鉴权

- 统一前缀：`/api/v1`
- 鉴权方式：`Authorization: Bearer <token>`
- 内容类型：`application/json`
- 文件上传：`multipart/form-data`
- 回调接口：按渠道协议要求，允许原始 body / form 提交

### 2.2 统一响应结构

```json
{
  "code": "0",
  "message": "OK",
  "data": {}
}
```

### 2.3 统一分页结构

```json
{
  "code": "0",
  "message": "OK",
  "data": {
    "list": [],
    "pageNo": 1,
    "pageSize": 20,
    "total": 0
  }
}
```

### 2.4 时间与金额约定

- 时间：统一 ISO-8601 字符串，如 `2026-03-16T10:30:00`
- 日期：统一 `yyyy-MM-dd`
- 金额：统一两位小数
- 水表读数：最多三位小数

### 2.5 错误码约定

| HTTP | code | 场景 |
|---|---|---|
| 400 | `INVALID_ARGUMENT` | 参数缺失 / 格式非法 |
| 401 | `UNAUTHORIZED` | 未登录 / Token 失效 |
| 403 | `FORBIDDEN` | 越权 / 数据范围不足 |
| 404 | `NOT_FOUND` | 资源不存在 |
| 409 | `CONFLICT` | 状态冲突，例如账单已支付、券已用 |
| 422 | `VALIDATION_FAILED` | 导入行校验失败 |
| 429 | `TOO_MANY_REQUESTS` | 高频点击 / 重复提交 |
| 500 | `INTERNAL_ERROR` | 系统异常 |

## 3. 角色与数据范围

### 3.1 角色

- `RESIDENT`：住户，仅访问自己绑定房间
- `AGENT`：经办人，仅访问授权用户组
- `ADMIN`：全局访问与配置
- `FINANCE`：全局或组织级报表 / 导出 / 对账

### 3.2 数据范围

- `Self`：本人绑定房间
- `Group`：Agent 授权用户组
- `All`：管理员全局

### 3.3 Token 建议载荷

```json
{
  "accountId": 10001,
  "accountType": "ADMIN",
  "roles": ["ADMIN"],
  "dataScope": "ALL",
  "groupIds": [5001, 5002],
  "exp": 1770000000
}
```

## 9. 支付回调接口

| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/callbacks/wechatpay` | 微信支付回调，验签/解密/幂等 |
| POST | `/callbacks/alipay` | 支付宝异步通知，验签/幂等 |

### 9.1 回调处理规则

1. 根据 `payOrderNo / out_trade_no` 查支付单。
2. 若 `pay_order.status = SUCCESS`，直接返回成功，禁止重复入账。
3. 若未成功，则事务内完成：
   - 更新 `pay_order = SUCCESS`
   - 更新 `bill = PAID`
   - 更新 `coupon_instance = USED`（如有）
   - 写入 `pay_transaction`
   - 触发支付后发券异步事件
4. 回调接口必须做到 **可重复调用但只推进一次状态**。

## 12. 联调注意事项

### 12.1 必做幂等

- `POST /payments` 必须带 `idempotencyKey`
- 回调必须按 `payOrderNo` 幂等
- 发券规则命中后，发券实例必须防重复

### 12.2 必做状态校验

- 账单已支付，禁止再次创建成功支付单
- 券已使用 / 已过期 / 已锁定，禁止重复抵扣
- 抄表读数 `currReading < prevReading` 直接拒绝
- 催缴只允许命中逾期未支付账单
- 发票申请只允许针对成功支付单创建

### 12.3 必做权限校验

- Resident 只能看自己绑定房间
- Agent 只能看授权 `groupId`
- Admin 才能进行开单、导入导出、配置类操作
- 电子凭证只允许 Resident 查看本人已支付账单对应的凭证，Admin 可做全局联调排查
- 发票申请处理只允许 Admin

### 12.4 必做前端处理

- 列表页必须有空态 / 加载态 / 错误态
- 支付结果页必须做轮询兜底
- 导入导出必须做异步任务状态轮询
- 新增凭证页时，必须在支付成功且 `voucherIssued=true` 时再展示入口

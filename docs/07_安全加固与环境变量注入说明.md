# 安全加固与环境变量注入说明

## 1. 本轮目标

本轮对后端做了上线前安全加固，重点解决以下问题：

- 微信登录不能再把前端 `code` 直接当作本地 `openId`
- 已签发 Token 需要支持立即失效
- 导入接口不能读取任意本地文件，也不能访问远程 URL
- AI Runtime API Key 不能明文落库
- 支付回调在并发和 Redis 退化场景下要保持幂等
- 优惠券秒杀在 Redis/RabbitMQ 开启和关闭两种场景下都要保持可用、可降级
- 生产敏感配置全部改为环境变量注入

## 2. 新增数据库迁移

本专项最初新增迁移脚本：

- `backend/src/main/resources/db/migration/V31__security_hardening.sql`

迁移内容：

- `account` 新增 `token_invalid_before`
- `coupon_instance` 新增奖励券唯一索引

唯一索引用途：

- 防止同一支付单在并发回调下重复发放奖励券

后续秒杀链路新增迁移脚本：

- `backend/src/main/resources/db/migration/V33__coupon_seckill.sql`

迁移内容：

- `coupon_seckill_campaign`
- `coupon_seckill_order`

用途：

- 记录秒杀活动、库存、每人限领、活动时间和状态
- 记录秒杀订单、请求号、处理状态、失败原因和最终发券结果
- 通过唯一约束保证同一居民同一请求号重复提交时幂等

## 3. 生产环境必须提供的 ENV

后端启动前，生产/预发环境至少需要注入以下变量：

```bash
APP_JWT_SECRET=

APP_BOOTSTRAP_ADMIN_USERNAME=admin
APP_BOOTSTRAP_ADMIN_PASSWORD=

APP_PAYMENT_WECHAT_MERCHANT_ID=
APP_PAYMENT_WECHAT_CALLBACK_SECRET=
APP_PAYMENT_ALIPAY_MERCHANT_ID=
APP_PAYMENT_ALIPAY_CALLBACK_SECRET=

APP_SECURITY_CRYPTO_CONFIG_ENCRYPTION_KEY=

APP_AUTH_WECHAT_MODE=real
APP_AUTH_WECHAT_APP_ID=
APP_AUTH_WECHAT_APP_SECRET=
APP_AUTH_WECHAT_JSCODE2SESSION_URL=https://api.weixin.qq.com/sns/jscode2session
```

可选但建议显式配置：

```bash
APP_IMPORT_EXPORT_IMPORT_DIR=
APP_IMPORT_EXPORT_EXPORT_DIR=

APP_INFRA_REDIS_ENABLED=
REDIS_HOST=
REDIS_PORT=
REDIS_PASSWORD=
REDIS_DB=
APP_INFRA_REDIS_KEY_PREFIX=
APP_INFRA_REDIS_CALLBACK_LOCK_SECONDS=
APP_INFRA_REDIS_SECKILL_LOCK_SECONDS=

APP_INFRA_RABBIT_ENABLED=
RABBITMQ_HOST=
RABBITMQ_PORT=
RABBITMQ_USERNAME=
RABBITMQ_PASSWORD=
RABBITMQ_VHOST=
APP_INFRA_RABBIT_PAYMENT_EXCHANGE=
APP_INFRA_RABBIT_PAYMENT_SUCCESS_ROUTING_KEY=
APP_INFRA_RABBIT_PAYMENT_SUCCESS_QUEUE=
APP_INFRA_RABBIT_PAYMENT_SUCCESS_DLQ=
APP_INFRA_RABBIT_COUPON_EXCHANGE=
APP_INFRA_RABBIT_COUPON_SECKILL_ROUTING_KEY=
APP_INFRA_RABBIT_COUPON_SECKILL_QUEUE=
APP_INFRA_RABBIT_COUPON_SECKILL_DLQ=

APP_AI_RUNTIME_API_KEY=
APP_AI_RUNTIME_API_BASE_URL=
APP_AI_RUNTIME_MODEL=
```

说明：

- `APP_JWT_SECRET` 未注入时，生产配置下后端不会正常启动
- `APP_BOOTSTRAP_ADMIN_PASSWORD` 用于初始化管理员登录口令；未注入时，默认种子管理员保持禁用
- `APP_SECURITY_CRYPTO_CONFIG_ENCRYPTION_KEY` 用于加密数据库中的 AI 密钥
- 微信登录生产默认应使用 `APP_AUTH_WECHAT_MODE=real`
- Redis 默认关闭；生产开启后用于支付回调锁、秒杀库存锁和 Agent 会话缓存
- RabbitMQ 默认关闭；生产开启后用于支付成功事件和优惠券秒杀订单事件

## 4. 行为变化

### 4.1 微信登录

- `mock` 模式下，测试和本地联调仍可继续使用 demo code
- `real` 模式下，后端会调用微信 `jscode2session`
- 只有换取到真实 `openId` 后，才会去匹配本地 `account_identity`

### 4.2 Token 失效

以下操作会立即让旧 Token 失效：

- 用户主动登出
- 账号被停用
- 管理员重置密码
- Agent 授权组被停用或收回

### 4.3 导入文件

导入仅允许读取 `app.import-export.import-dir` 目录内的本地文件。

明确拒绝：

- `http://`
- `https://`
- `file:`
- 越界绝对路径
- 相对路径逃逸

### 4.4 AI Runtime 配置

- 数据库保存时对 `apiKey` 做加密
- 读取时自动解密
- 展示层只返回脱敏后的 `apiKeyMasked`
- 兼容历史明文数据读取

### 4.5 支付回调

- 支付单成功更新改为 CAS
- 同时使用数据库行锁串行化支付回调处理
- 奖励券以数据库唯一索引兜底防重

### 4.6 优惠券秒杀

- 秒杀提交使用 `requestId` 做请求幂等
- Redis 开启时使用 Redisson 锁保护活动库存扣减
- RabbitMQ 开启时秒杀订单进入队列异步处理
- RabbitMQ 关闭时服务内同步处理，保证本地开发、测试和降级场景仍可运行
- 库存不足、活动未开始/已结束、超过每人限领都会写入订单失败原因

## 5. 测试覆盖

本轮补充并通过了以下回归：

- 登出后旧 access / refresh token 失效
- 账号停用后旧 token 失效
- Agent 收权后旧 token 立即失去数据范围
- AI Runtime API Key 入库为密文
- 导入接口拒绝远程 URL 与越界路径
- 支付重复回调不会重复发券
- 优惠券秒杀重复请求返回同一订单，不重复发券
- 优惠券秒杀库存售罄时订单失败且库存不为负
- Agent 调用链路相关原有测试全部继续通过

## 6. 发布注意事项

- 发布前先执行 Flyway，确保 `V31__security_hardening.sql` 已生效
- 若发布包含优惠券秒杀能力，还需确保 `V33__coupon_seckill.sql` 已生效
- 部署平台要先补齐 ENV，再启动后端
- 若生产库历史上已存在人工写入的明文 AI Key，本版仍可读取，但建议后续通过管理端重新保存一次，完成加密重写
- Redis/RabbitMQ 开启前要完成连通性、权限、队列、死信队列、告警和降级演练

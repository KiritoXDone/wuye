## 2. 当前冻结业务口径

- 房间是账单主体，不是账号
- 一个房间可以绑定多个居民账号
- 物业费按面积计费、按年开单、按年缴纳
- 水费按月抄表，录入后生成月度水费账单
- 账单按费用类型分单
- 报表默认按房间统计
- 已支付账单不能直接覆盖
- 支付回调、奖励券发放和秒杀领券必须幂等
- 高风险 Agent 动作必须经过确认后执行

## 3. 当前冻结技术口径

- 后端：Java 17、Spring Boot 3.3、MyBatis、MySQL 8、Flyway、Maven
- 可选基础设施：Redis/Redisson、RabbitMQ
- 管理端：React 18、TypeScript、Tailwind CSS、Vite
- 小程序端：原生微信小程序、TypeScript
- 管理端发布：Nginx 静态托管，需支持快速回滚
- 后端发布：先执行数据库迁移，再启动应用

## 5. 本次同步能力

### AI 批量账单动作

- `PROPERTY_BILL_GENERATE`：生成指定小区指定年度物业费账单
- `WATER_BILL_GENERATE`：生成指定小区指定年月水费账单
- 两类动作均属于 L4 高风险动作，需先预览参数，再携带确认 token 执行
- 执行结果会回写摘要和 Markdown 明细，供管理端和小程序端展示

### 优惠券秒杀

- 管理端接口：`POST /api/v1/admin/coupons/seckill-campaigns`
- 居民提交：`POST /api/v1/coupons/seckill/{campaignId}/orders`
- 订单查询：`GET /api/v1/coupons/seckill/orders/{orderNo}`
- 数据表：
  - `coupon_seckill_campaign`
  - `coupon_seckill_order`
- Redis 开启时使用 Redisson 锁保护秒杀库存扣减
- RabbitMQ 开启时使用异步队列处理秒杀订单
- RabbitMQ 未开启时服务内同步处理，保证本地开发和测试可运行

### 支付幂等测试收口

- 支付回调重复处理不会重复发券
- 居民账单列表按可见账单过滤，未来账单测试改为直接核对数据库生成结果
- 测试清理覆盖秒杀活动和秒杀订单表，避免集成测试相互污染

## 7. 注意事项

- 不再以旧的“月度物业费 + Vue 管理端”口径作为实现目标
- 当前“手动标记已缴”仅支持整单确认，不支持部分缴费拆分
- Redis/RabbitMQ 默认关闭；生产开启前要完成连接、队列、权限、告警和降级演练
- AI Runtime 默认关闭；开启前必须注入 API Key 加密密钥，并确认供应商、模型和超时配置
- 若后续扩展线下缴费导入、批量确认已缴或差额补收，需要补充迁移与审计说明

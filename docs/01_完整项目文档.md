# 物业管理系统完整项目文档

## 1. 项目定位

本项目是面向住宅物业收费和居民服务的物业管理系统。系统以房间为核心主体，围绕年度物业费、月度水费、居民缴费、优惠券、催缴、发票、审计和 AI 助手形成闭环。

当前工程不是原型页面集合，而是按真实上线交付口径实现：

- 后端接口可运行
- 数据库迁移可追踪
- 关键业务有集成测试
- 管理端和小程序端有真实页面承接
- Redis、RabbitMQ、AI Runtime 等基础设施默认可降级，生产可显式开启

## 2. 用户角色

### Admin

物业后台操作人员。主要负责：

- 维护小区、房型、房间、账号和授权组
- 配置物业费、水费规则
- 生成年度物业费账单和月度水费账单
- 查看缴费统计、账单、报表、审计日志
- 处理催缴、发票、优惠券、线下缴费确认
- 使用 AI 助手执行高风险业务动作

### Resident

居民小程序用户。主要负责：

- 微信登录
- 绑定或确认房间
- 查看房间账单和账单详情
- 创建支付单并轮询支付状态
- 查看可用优惠券和兑换券
- 参加优惠券秒杀活动
- 使用智能助手查询账单、发起支付等操作

## 3. 核心业务口径

- 房间是账单主体，不是账号。
- 多个居民账号可以绑定同一个房间。
- 一个房间同一年只能有一张有效物业费账单。
- 一个房间同一年月只能有一张有效水费账单。
- 水费按表计读数计算，录入读数后生成当月账单。
- 已支付账单不能直接覆盖。
- 报表、欠费户数、缴费率默认按房间统计。
- 支付回调必须支持重复回调幂等。
- 优惠券奖励、手工发券、秒杀发券必须防重复。
- Agent 执行会改变数据的动作前必须先预览并确认。

## 4. 技术架构

```text
微信小程序 Resident 端
        |
        | HTTPS / JSON
        v
Spring Boot API  ---- MySQL 8 / Flyway
        |
        | 可选
        +---- Redis / Redisson
        |
        | 可选
        +---- RabbitMQ
        |
        | 可选
        +---- AI Runtime Provider

React Web 管理端
        |
        | HTTPS / JSON / SSE
        v
Spring Boot API
```

### 后端分层

- `controller`：HTTP API、参数校验、统一响应
- `service`：业务编排、幂等、权限和审计边界
- `mapper`：MyBatis SQL
- `entity/dto/vo`：数据库实体、入参对象、响应对象
- `common`：鉴权、异常、基础设施、配置
- `db/migration`：Flyway 数据库迁移

### 前端结构

- `frontend/admin-web`：管理端 React 单页应用
- `frontend/miniapp-wechat`：微信小程序 Resident 端

## 8. 基础设施与降级

### Redis / Redisson

Redis 默认关闭：

```bash
APP_INFRA_REDIS_ENABLED=false
```

开启后用于：

- 支付回调锁
- 秒杀库存锁
- Agent 会话缓存

未开启时：

- 支付幂等依赖数据库行锁/CAS 和唯一索引
- 秒杀库存锁使用空实现，本地开发和测试仍可运行
- Agent 会话缓存使用空实现

### RabbitMQ

RabbitMQ 默认关闭：

```bash
APP_INFRA_RABBIT_ENABLED=false
```

开启后用于：

- 支付成功事件
- 优惠券秒杀订单事件

未开启时：

- 支付主流程不依赖 MQ 成功
- 优惠券秒杀订单在服务内同步处理

## 14. 已知边界

- 真实支付渠道仍需接入正式签名验签、证书轮换、退款和对账。
- 优惠券秒杀当前已有后端链路和测试，管理端活动列表、编辑、下线页面仍可继续完善。
- 当前线下手动标记已缴按整单确认，不支持部分缴费。
- Nginx 静态包回滚脚本和生产演练记录仍需补齐。
- AI Runtime 可配置供应商和模型，但生产使用前需要确定供应商限流、审计和内容安全策略。
